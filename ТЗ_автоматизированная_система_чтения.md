2 Цели и назначение создания автоматизированной системы

2.1 Цель создания автоматизированной системы

Целью создания автоматизированной системы является разработка интерактивной платформы для чтения электронных книг, предназначенной для взрослых пользователей и детей, обеспечивающей организацию ежедневного чтения, контроль хода чтения, проверку усвоения прочитанного материала и поддержку родительского контроля.

Создаваемая система должна предоставлять пользователю возможность работать с электронными книгами в цифровом формате, вести учет прогресса чтения, фиксировать время, затрачиваемое на чтение страниц, а также формировать контрольные вопросы по прочитанному фрагменту. Для сценария использования с участием ребенка система должна предоставлять родителю средства назначения материалов для чтения и анализа результатов ребенка.

2.2 Назначение создания автоматизированной системы

Назначением автоматизированной системы является повышение эффективности и качества чтения за счет применения средств контроля, мотивации и аналитики.

Система предназначена для решения следующих задач:

1. организации регулярного чтения пользователями электронных книг;
2. контроля факта чтения и предотвращения формального пролистывания страниц;
3. проверки понимания прочитанного текста с использованием тестовых вопросов;
4. накопления и отображения статистики чтения;
5. предоставления родителям инструментов наблюдения за читательской активностью детей.

3 Характеристика объектов автоматизации

3.1 Объект автоматизации

Объектом автоматизации является процесс чтения электронных книг, включая организацию чтения по частям, контроль выполнения дневной нормы чтения, фиксацию прогресса пользователя, формирование статистики и реализацию механизмов родительского контроля.

Автоматизации подлежат следующие процессы:

1. регистрация и авторизация пользователей;
2. загрузка, хранение и обработка электронных книг;
3. чтение книг с фиксацией времени и текущего положения;
4. автоматическое формирование тестов по прочитанному материалу;
5. оценка результатов тестирования;
6. накопление статистики чтения;
7. взаимодействие между пользователями в рамках связей "родитель-ребенок" и иных пользовательских связей;
8. контроль чтения ребенка со стороны родителя.

3.2 Характеристика пользователей системы

В системе предусматриваются следующие категории пользователей:

1. обычный пользователь;
2. родитель;
3. ребенок.

Обычный пользователь самостоятельно загружает материалы для чтения, устанавливает дневную цель, читает книгу по частям, проходит тестирование после прочтения фрагмента и получает статистику собственной активности.

Пользователь, выполняющий роль родителя, получает дополнительные функциональные возможности, связанные с контролем чтения ребенка. Родитель может устанавливать связь с ребенком, загружать для него книги и просматривать сведения о ходе чтения и результатах тестирования.

Пользователь, выполняющий роль ребенка, читает назначенные или доступные ему книги, проходит контрольные тесты и формирует статистику, доступную для просмотра родителю в пределах предоставленных системой прав доступа.

3.3 Характеристика автоматизируемого процесса

В общем случае процесс работы системы организуется следующим образом.

Пользователь загружает электронную книгу в систему и задает дневную цель чтения. После обработки материала система предоставляет книгу для постраничного чтения. Ход чтения фиксируется автоматически: сохраняется текущая страница, учитывается затраченное время, рассчитываются показатели прогресса.

После прочтения установленного фрагмента текста система формирует контрольные вопросы для проверки качества усвоения материала. При успешном прохождении теста пользователь получает возможность продолжить чтение следующего фрагмента. При неудовлетворительном результате система может потребовать повторного ознакомления с уже прочитанным фрагментом.

Для сценария с участием родителя и ребенка процесс дополняется контролирующими действиями со стороны родителя. Родитель назначает ребенку материалы, отслеживает статистику чтения, анализирует результаты тестирования и контролирует выполнение дневной цели.

4 Требования к автоматизированной системе

4.1 Требования к автоматизированной системе в целом

4.1.1 Требования к структуре системы

Автоматизированная система должна иметь клиент-серверную архитектуру и включать:

1. серверную часть, обеспечивающую хранение данных, обработку запросов, реализацию бизнес-логики и формирование статистики;
2. клиентскую веб-часть, обеспечивающую доступ пользователей через браузер;
3. клиентскую мобильную часть, обеспечивающую доступ пользователей через мобильное приложение;
4. базу данных, предназначенную для хранения пользовательских данных, книг, параметров чтения, результатов тестирования и статистических показателей.

Система должна обеспечивать согласованную работу серверной и клиентских частей.

4.1.2 Требования к режимам функционирования

Система должна обеспечивать функционирование в следующих основных режимах:

1. режим регистрации и аутентификации пользователей;
2. режим работы с библиотекой книг;
3. режим чтения книги;
4. режим тестирования по прочитанному материалу;
5. режим просмотра статистики;
6. режим родительского контроля;
7. режим администрирования и сопровождения данных.

4.2 Требования к функционалу автоматизированной системы

4.2.1 Требования к функционалу платформы

Автоматизированная система должна обеспечивать выполнение следующих общесистемных функций:

1. формирование и отправку уведомлений о необходимости чтения в соответствии с дневной целью пользователя;
2. автоматический учет серии дней чтения пользователя;
3. автоматическую генерацию контрольных вопросов по прочитанному фрагменту текста;
4. хранение и обработку информации о книгах, сессиях чтения и результатах тестирования;
5. формирование статистических показателей по чтению;
6. разграничение доступа к функциям системы в зависимости от роли пользователя;
7. поддержку работы в веб-версии и мобильной версии.

4.2.2 Требования к функционалу учетной записи пользователя

Система должна обеспечивать выполнение следующих функций управления учетной записью:

1. регистрацию нового пользователя;
2. проверку корректности вводимых регистрационных данных;
3. подтверждение адреса электронной почты пользователя;
4. авторизацию пользователя;
5. завершение пользовательской сессии путем выхода из учетной записи;
6. смену пароля пользователя;
7. хранение и использование данных авторизации в рамках активной сессии.

4.2.3 Требования к функционалу пользователя

Для пользователя, работающего с системой в обычном режиме, система должна обеспечивать:

1. загрузку электронных книг в систему;
2. поддержку допустимых форматов электронных книг;
3. задание дневной цели чтения при добавлении книги;
4. формирование личной библиотеки пользователя;
5. просмотр списка загруженных книг;
6. просмотр сведений о книге и текущем прогрессе чтения;
7. изменение дневной цели чтения;
8. удаление книги из личной библиотеки;
9. чтение книги в постраничном режиме;
10. сохранение текущего положения в книге;
11. продолжение чтения с последнего сохраненного места;
12. фиксацию времени чтения страницы;
13. ограничение перехода к следующей странице при несоблюдении минимальных условий чтения;
14. прохождение тестирования после прочтения установленного фрагмента текста;
15. повторное чтение фрагмента в случае неудовлетворительного результата тестирования;
16. просмотр личной статистики чтения;
17. просмотр информации о серии дней чтения;
18. просмотр рейтинга пользователей по читательской активности;
19. просмотр входящих уведомлений и запросов.

4.2.4 Требования к функционалу родителя

Для пользователя, выполняющего роль родителя, система должна дополнительно обеспечивать:

1. отправку запроса на установление связи с другим пользователем в роли ребенка;
2. просмотр списка связанных детей;
3. подтверждение, отклонение и прекращение связи с ребенком в рамках предусмотренной логики системы;
4. загрузку книг в библиотеку ребенка;
5. задание или изменение дневной цели чтения для книг ребенка;
6. просмотр списка книг ребенка;
7. просмотр статистики чтения ребенка;
8. просмотр детальной статистики по каждой книге ребенка;
9. просмотр результатов тестирования ребенка;
10. просмотр сведений о времени, затрачиваемом ребенком на чтение;
11. контроль выполнения ребенком установленной дневной цели.

4.2.5 Требования к функционалу ребенка

Для пользователя, связанного с родителем в роли ребенка, система должна обеспечивать:

1. доступ к книгам, назначенным родителем;
2. чтение назначенных материалов в соответствии с установленной дневной целью;
3. прохождение тестирования по прочитанным фрагментам;
4. просмотр собственного прогресса чтения;
5. просмотр собственной статистики чтения в пределах предоставленных прав доступа;
6. получение и обработку запросов на установление связи с родителем;
7. выполнение ограничений на изменение или удаление материалов, назначенных родителем, если такие ограничения заданы логикой системы.

4.2.6 Требования к функциям взаимодействия пользователей

Система должна обеспечивать:

1. отправку запросов на установление пользовательских связей;
2. просмотр входящих пользовательских запросов;
3. подтверждение или отклонение входящих запросов;
4. просмотр списка установленных связей;
5. прекращение ранее установленных связей в соответствии с правами доступа пользователя.

4.2.7 Требования к статистическим и аналитическим функциям

Система должна обеспечивать автоматическое формирование и отображение следующих аналитических показателей:

1. общее количество прочитанных страниц;
2. общее время чтения;
3. среднее время чтения одной страницы;
4. скорость чтения;
5. количество книг, находящихся в процессе чтения;
6. количество завершенных книг;
7. количество пройденных тестов;
8. количество успешно пройденных тестов;
9. средний результат тестирования;
10. степень выполнения дневной цели;
11. текущая серия дней чтения;
12. максимальная серия дней чтения;
13. рейтинг пользователей по показателям читательской активности.

4.2.8 Требования к ограничениям выполнения функций

При реализации функционала системы должны соблюдаться следующие условия:

1. доступ к защищенным функциям системы должен предоставляться только авторизованным пользователям;
2. вход в систему должен быть доступен только после подтверждения адреса электронной почты;
3. продолжение чтения должно зависеть от выполнения условий, установленных для текущего этапа чтения;
4. учет выполнения дневной цели должен осуществляться автоматически на основании фактически зафиксированного чтения;
5. результаты тестирования должны учитываться системой при принятии решения о возможности продолжения чтения;
6. доступ родителя к данным ребенка должен предоставляться только при наличии подтвержденной связи между соответствующими учетными записями.

4.3 Требования к обеспечению автоматизированной системы

4.3.1 Требования к программному обеспечению автоматизированной системы

4.3.1.1 Требования к программному обеспечению веб-версии автоматизированной системы

Для реализации веб-версии системы должно использоваться следующее программное обеспечение:

1. операционная система сервера: Ubuntu 22.04 LTS;
2. язык программирования серверной части: Python 3.10 или выше;
3. веб-фреймворк: Django версии 4.x или выше;
4. средства рендеринга интерфейса веб-версии: Django Templates или аналогичный шаблонизатор;
5. средство взаимодействия с базой данных: Django ORM;
6. система управления базами данных: PostgreSQL 14 или выше либо иная реляционная СУБД с аналогичными характеристиками;
7. веб-сервер приложений: Gunicorn;
8. обратный прокси-сервер: Nginx;
9. средство аутентификации и авторизации: встроенные механизмы Django с возможностью расширения средствами OAuth 2.0;
10. средства контейнеризации: Docker;
11. сервис push-уведомлений: Firebase Cloud Messaging;
12. средство документирования API: OpenAPI/Swagger или аналогичное решение;
13. средство организации виртуального окружения Python: venv.

4.3.1.2 Требования к программному обеспечению мобильной версии автоматизированной системы

Для реализации мобильной версии системы должно использоваться следующее программное обеспечение:

1. среда разработки: Android Studio;
2. язык разработки: Kotlin или Java;
3. средства выполнения сетевых запросов: Retrofit и OkHttp;
4. сервис push-уведомлений: Firebase Cloud Messaging;
5. минимальная поддерживаемая версия операционной системы Android: Android 10, API 29 и выше.

4.3.2 Требования к информационному обеспечению

Информационное обеспечение системы должно включать:

1. данные учетных записей пользователей;
2. сведения об электронных книгах;
3. данные о связях между пользователями;
4. данные о сессиях чтения;
5. результаты тестирования;
6. статистику чтения и аналитические показатели;
7. служебные данные, необходимые для функционирования уведомлений, авторизации и хранения пользовательских настроек.

4.3.3 Требования к надежности

Система должна обеспечивать:

1. сохранность пользовательских данных при штатной эксплуатации;
2. корректное восстановление работы после кратковременных сбоев клиентской части;
3. сохранение текущего прогресса чтения пользователя;
4. обработку ошибочных действий пользователя с выдачей понятных сообщений;
5. защиту от потери данных при завершении пользовательской сессии.

4.3.4 Требования к защите информации

Система должна обеспечивать:

1. идентификацию и аутентификацию пользователей;
2. разграничение доступа к данным в соответствии с ролью пользователя;
3. защиту данных учетной записи пользователя;
4. защиту персональных данных пользователей в пределах, предусмотренных назначением системы;
5. недопущение доступа родителя к данным пользователя, не связанного с ним в роли ребенка;
6. недопущение доступа посторонних лиц к данным пользователя без прохождения процедуры аутентификации.

6 Порядок разработки автоматизированной системы

Исходными данными для разработки автоматизированной системы являются:

1. техническое задание;
2. материалы научно-исследовательской части;
3. результаты анализа предметной области;
4. требования к структуре и функционалу системы.

Разработка системы должна выполняться поэтапно, с последовательным уточнением требований, реализацией функциональных модулей и проведением проверки их работоспособности.

По окончании соответствующих этапов работ должны быть подготовлены следующие материалы:

1. расчетно-пояснительная записка;
2. схемы и диаграммы, отражающие архитектуру и алгоритмы работы системы;
3. программная реализация серверной, веб- и мобильной частей;
4. документация пользователя и разработчика.

7 Требования к документированию

По окончании разработки автоматизированной системы должны быть подготовлены следующие документы:

1. руководство программиста;
2. руководство администратора, содержащее описание порядка развертывания и запуска системы;
3. руководство пользователя;
4. расчетно-пояснительная записка;
5. графические материалы, отражающие архитектуру и основные алгоритмы функционирования системы.

Документация должна быть оформлена в соответствии с требованиями, предъявляемыми к документам, сопровождающим разработку автоматизированных систем в рамках учебного проекта.
